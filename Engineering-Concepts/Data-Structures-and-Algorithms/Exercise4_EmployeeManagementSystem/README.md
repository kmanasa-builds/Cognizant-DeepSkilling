# Exercise 4 - Employee Management System

## Objective
Develop an Employee Management System using arrays to efficiently manage employee records.

## Scenario
A company needs a system to store and manage employee information. Arrays are used to store employee records and perform various operations.

## Employee Attributes
- Employee ID
- Name
- Position
- Salary

## Operations Implemented

### Add Employee
Adds a new employee record to the array.

### Search Employee
Searches for an employee using Employee ID.

### Traverse Employees
Displays all employee records stored in the array.

### Delete Employee
Removes an employee record from the array and shifts remaining elements.

## Time Complexity Analysis

| Operation | Time Complexity |
|-----------|----------------|
| Add | O(1) |
| Search | O(n) |
| Traverse | O(n) |
| Delete | O(n) |

## Advantages of Arrays
- Fast access using index.
- Easy to implement.
- Efficient for fixed-size datasets.

## Limitations of Arrays
- Fixed size.
- Insertion and deletion can be costly.
- Memory may be wasted if array size is larger than required.

## Conclusion
Arrays are suitable for managing a small to moderate number of employee records where direct access and simple operations are required.
