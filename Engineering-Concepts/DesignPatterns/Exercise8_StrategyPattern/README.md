# Exercise 8 - Strategy Pattern

## Objective

Implement the Strategy Design Pattern to allow selecting different payment methods at runtime.

## Classes

- PaymentStrategy (Strategy Interface)
- CreditCardPayment (Concrete Strategy)
- PayPalPayment (Concrete Strategy)
- UpiPayment (Concrete Strategy)
- PaymentContext (Context)
- Main (Test Class)

## Output

Paid Rs.2500.0 using Credit Card.
Paid Rs.1800.0 using PayPal.
Paid Rs.1200.0 using UPI.

## Strategy Pattern

The Strategy Pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. It allows the behavior of an object to be selected at runtime without modifying its code.
